Before We Start

What You Will Learn

This tutorial is based on a real, professional HTML document. Every technique here was used in an actual project — not a toy example. You will learn how to build a dark-themed, multi-section page with its own design language: color system, typography, cards, tables, animations, and more.

By the end, you will understand how all of these pieces connect into a single cohesive design. Everything is written in plain HTML and CSS — no frameworks, no libraries required.

Who Is This For? Students who already know basic HTML tags like <div>, <p>, <h1> and want to learn how professional pages are actually styled and structured.
Lesson 1

CSS Variables — Your Design Tokens

CSS variables (also called custom properties) let you define a value once and reuse it everywhere. You define them inside :root, which means they apply to the whole page. 

/* Define your variables once */
:root {
  --bg:      #0b0b0e;
  --primary: #7c3aed;
  --text:    #cdc8d8;
}

/* Use them anywhere with var() */
body {
  background: var(--bg);
  color:      var(--text);
}

.card {
  border: 1px solid var(--primary);
}
Why This Matters If you want to change your accent color from violet to red, you only edit one line in :root. Without variables, you would search-and-replace that color across hundreds of lines.

Your Color Palette Demo

Rule of Thumb Dark themes need at least 3 background shades (bg, surface, panel), 1 accent color, its dim/glow variants, and 2–3 text shades. That's roughly 10–15 variables covering your whole page.
Lessons 2 & 3

Reset & Body Setup

The Universal Reset

Every browser applies its own default styles. A reset removes them so you start from a clean slate.

*, *::before, *::after {
  box-sizing: border-box; /* padding stays inside the box */
  margin:     0;              /* no default spacing */
  padding:    0;
}
box-sizing: border-box Without this, if you set width: 300px and add padding: 20px, the box becomes 340px wide. With border-box, it stays 300px — the padding eats into it. This is almost always what you want.

Body as a Canvas

The body sets your page's default font, text color, background, and overall feel. You can stack multiple background layers using commas.

body {
  background:  var(--bg);
  color:       var(--text);
  font-family: 'Crimson Pro', Georgia, serif;
  font-size:   17px;
  line-height: 1.8;
  min-height:  100vh; /* at least full screen height */

  /* TWO background layers stacked: */
  background-image:
    /* 1. Radial glow at the top */
    radial-gradient(ellipse 70% 40% at 50% -10%,
      rgba(124,58,237,0.08) 0%, transparent 60%),
    /* 2. Subtle vertical grid lines */
    repeating-linear-gradient(
      90deg,
      transparent 0px,
      transparent 79px,
      rgba(30,20,40,0.2) 80px
    );
}
background-image layers Comma-separated backgrounds stack on top of each other. The first one in the list is on top. You can combine gradients, textures, and even images this way.
Lesson 4

Typography System

Professional pages use multiple fonts for different roles. Load them from Google Fonts with a <link> tag in your <head>.

<!-- In your <head> -->
<link href="https://fonts.googleapis.com/css2?family=Bebas+Neue
  &family=Share+Tech+Mono
  &family=Crimson+Pro:ital,wght@0,400;1,300
  &display=swap" rel="stylesheet">
FontRoleExample Use
Bebas NeueDisplay / Titlesh1, h2, h3 headings
Share Tech MonoUI Labels / Codebadges, section labels, table headers, code
Crimson ProBody / Readingparagraphs, descriptions, long text

clamp() — Responsive Font Sizes

.title {
  font-size: clamp(38px, 6vw, 72px);
  /*         MIN    ↑     MAX
             never  scales  never
             below  with    above
             38px   screen  72px  */
}

Letter Spacing & Text Transform

For uppercase mono labels, letter-spacing and text-transform: uppercase are what give them that "military UI" look.

.section-label {
  font-family:     'Share Tech Mono', monospace;
  font-size:       10px;
  letter-spacing:  4px;
  text-transform:  uppercase;
  color:           var(--primary);
}
Font Pairing Rule One serif or slab for body → readable at small sizes. One condensed display for headings → bold impact. One monospace for UI text → technical and precise. Three fonts is usually the maximum before it feels chaotic.
Lesson 5

Page Layout & Wrapper

Center your content with a max-width wrapper. This prevents text from stretching too wide on large screens.

.wrapper {
  position:   relative;
  z-index:    1;          /* above body::before overlay */
  max-width:  980px;
  margin:     0 auto;     /* center horizontally */
  padding:    0 28px 80px; /* no top, sides, big bottom */
}
margin: 0 auto This is the classic centering trick. 0 = no top/bottom margin. auto = browser divides remaining space equally on both sides → centered.

Section Structure Pattern

Every major section follows the same HTML structure. This consistency makes your page feel like a real product:

<section class="section" id="my-section">

  <!-- Small label above the heading -->
  <div class="section-label">Chapter One</div>

  <!-- Main section heading -->
  <h2>My Section Title</h2>

  <!-- Content goes here -->
  <p>Your content...</p>

</section>
Lesson 6

Hero Section

The hero is the first thing visitors see. It needs to be big, confident, and atmospheric. It typically contains: an eyebrow label, a large title, a subtitle, and badges. 

<header class="hero">

  <!-- Small text above the title -->
  <div class="eyebrow">Addon Documentation</div>

  <!-- Big display title -->
  <h1 class="lesson-title">
    Cultivator's<br>
    <span>Anarchy</span> <!-- accent color -->
  </h1>

  <!-- Feature badges -->
  <div class="badges">
    <span class="badge purple">Feature One</span>
    <span class="badge">Feature Two</span>
  </div>

</header>

The Glowing Bottom Border Trick

The hero has a gradient line at the bottom using ::after. This is much more elegant than a plain border-bottom.

.hero {
  position: relative; /* needed for ::after positioning */
  overflow: hidden;
}
.hero::after {
  content:    '';
  position:   absolute;
  bottom: 0; left: -28px; right: -28px;
  height:     1px;
  background: linear-gradient(90deg,
    transparent, var(--primary), transparent);
}
Lesson 7

Pseudo-Elements

::before and ::after insert invisible "ghost" elements before or after any element's content. They must have content: '' (even if empty) to appear. They're used for decorative lines, overlays, icons, and stripes.

::before
Inserts a ghost element before the content. Used for: decorative lines, icons that prefix text, the line in our eyebrow labels.
.eyebrow::before {
  content: '';
  width:   28px;
  height:  1px;
  background: var(--primary);
  display: inline-block;
}
::after
Inserts a ghost element after the content. Used for: gradient lines, extending decorations, overlays.
.section-label::after {
  content:    '';
  flex:        1;
  height:      1px;
  background:  linear-gradient(
    90deg,
    var(--primary-dim),
    transparent);
}
Critical Rule A pseudo-element that uses position: absolute requires its parent to have position: relative. Otherwise it will position relative to the whole page. This is one of the most common beginner mistakes.

The Scanline Overlay

The subtle horizontal stripes across the whole page are a body::before with position: fixed so they stay on screen at all times.

body::before {
  content:         '';
  position:        fixed;
  inset:           0;       /* same as top:0 right:0 bottom:0 left:0 */
  pointer-events:  none;   /* doesn't block mouse clicks */
  z-index:         0;
  background: repeating-linear-gradient(
    0deg,
    transparent,
    transparent 2px,
    rgba(0,0,0,0.035) 2px,
    rgba(0,0,0,0.035) 4px
  );
}
Lesson 8

Cards & Callout Boxes

Cards

Cards group related content. The top accent stripe is done with ::before. Variants (gold, green, dim) just override that one gradient.

.card {
  background:    var(--panel);
  border:        1px solid var(--border);
  border-radius: 4px;
  padding:       22px 26px;
  position:      relative; /* for ::before */
  overflow:      hidden;   /* clip the ::before stripe */
}
.card::before {
  content: '';
  position: absolute;
  top: 0; left: 0; right: 0;
  height: 2px;
  background: linear-gradient(90deg, var(--primary), transparent);
}
/* Gold variant — just override the gradient */
.card.gold::before {
  background: linear-gradient(90deg, var(--gold), transparent);
}

Callout Boxes

Callouts use a border-left flag instead of a top stripe. Better for short inline notes.

.callout {
  padding:     14px 18px;
  border-left: 2px solid var(--primary);
  background:  var(--primary-soft);
  border-radius: 3px;
}
Violet CalloutFor tips, important notes, and key concepts.
Gold CalloutFor step-by-step instructions or featured tips.
Green CalloutFor success messages or positive information.
Dim CalloutFor secondary notes, caveats, or technical clarifications.
Lesson 9

Code Blocks

Two types: inline code for short snippets within text, and <pre><code> blocks for multi-line examples.

/* Inline code */
code {
  font-family:   'Share Tech Mono', monospace;
  font-size:     13px;
  color:         var(--primary);
  background:    var(--code-bg);
  padding:       1px 7px;
  border-radius: 2px;
  border:        1px solid var(--border);
}

/* Multi-line code block */
pre {
  background:    var(--code-bg);
  border:        1px solid var(--border);
  border-radius: 4px;
  padding:       20px 22px;
  overflow-x:    auto; /* horizontal scroll if too wide */
}
/* Language label in top-right using data attribute */
pre::before {
  content:  attr(data-lang); /* reads HTML attribute */
  position: absolute;
  top: 10px; right: 14px;
  font-size: 9px; color: var(--text-dim);
}
attr() Trick content: attr(data-lang) reads the data-lang="CSS" attribute from your HTML and uses it as the label text. This is how the language tags (CSS, HTML) appear in the top-right of every code block on this page!
Lesson 10

Tables

Tables are for structured data with rows and columns. Always wrap them in a .table-wrap div so they scroll horizontally on mobile instead of breaking the layout.

<div class="table-wrap">
  <table>
    <thead>
      <tr>
        <th>Column A</th>
        <th>Column B</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>Row 1 A</td>
        <td class="t-good">Row 1 B</td>
      </tr>
    </tbody>
  </table>
</div>

Lesson 11

CSS Grid Layouts

CSS Grid is the easiest way to create multi-column layouts. Define columns with grid-template-columns and space between them with gap.

/* Two equal columns */
.two-col {
  display:               grid;
  grid-template-columns: 1fr 1fr;
  gap:                   14px;
}

/* Three equal columns */
.three-col {
  display:               grid;
  grid-template-columns: 1fr 1fr 1fr;
  gap:                   12px;
}

/* Collapse to 1 column on small screens */
@media (max-width: 620px) {
  .two-col {
    grid-template-columns: 1fr;
  }
}

Live Three-Column Example

Column One
Each of these cards sits in a grid column automatically.
Column Two
No floats, no inline-block. Just display: grid.
Column Three

1fr Explained fr means "fraction of available space." 1fr 1fr = two equal halves. 2fr 1fr = left column is twice as wide as the right.
Lessons 12 & 13

Animations & Transitions

@keyframes Animations

Define an animation with @keyframes, then apply it with the animation property.

/* 1. Define the keyframes */
@keyframes fadeUp {
  from { opacity: 0; transform: translateY(18px); }
  to   { opacity: 1; transform: translateY(0); }
}

/* 2. Apply the animation */
.hero {
  animation: fadeUp 0.7s ease both;
  /*         name  dur  easing fill-mode */
}

/* Staggered delay for sections */
.section {
  animation:       fadeUp 0.5s ease both;
  animation-delay: 0.1s; /* starts 0.1s later */
}

Transition — Smooth State Changes

transition makes CSS property changes animate smoothly instead of snapping instantly. Used for hover effects.

.hover-card {
  border:     1px solid var(--border);
  transform:  translateY(0);

  /* Animate these 3 properties smoothly */
  transition: border-color 0.2s,
               background   0.2s,
               transform    0.15s;
}

.hover-card:hover {
  border-color: var(--primary-dim);
  background:   #17171e;
  transform:    translateY(-2px); /* lifts slightly */
}
EasingFeelBest For
easeSlow start, fast middle, slow endMost UI transitions
ease-outFast start, slows to stopElements entering screen
ease-inSlow start, fast endElements leaving screen
linearConstant speedLooping animations, spinners
Lesson 14

Fixed Sidebar & Scroll Script

The dots on the left side of this page are a fixed navigation panel. They stay visible as you scroll and highlight to show your current position.

HTML Structure

<nav class="toc">
  <a class="toc-dot active" href="#intro" title="Introduction"></a>
  <a class="toc-dot"        href="#vars"  title="Variables"></a>
  <!-- one dot per section -->
</nav>

CSS

.toc {
  position:  fixed;         /* stays on screen */
  top:       50%;
  left:      18px;
  transform: translateY(-50%); /* true vertical center */
  display:   flex;
  flex-direction: column;
  gap:       7px;
  z-index:   100;
}
.toc-dot.active {
  background:  var(--primary);
  box-shadow:  0 0 8px var(--primary); /* glow */
}
/* Hide on small screens (no room) */
@media (max-width: 1100px) {
  .toc { display: none; }
}

JavaScript — Scroll Detection

// Get all dot links and all sections
var dots     = document.querySelectorAll('.toc-dot');
var sections = document.querySelectorAll('section[id]');

window.addEventListener('scroll', function() {
  var scrollY  = window.scrollY + 200; // offset so it triggers early
  var current  = '';

  // Find which section we're currently in
  sections.forEach(function(s) {
    if (s.offsetTop <= scrollY) current = s.id;
  });

  // Toggle 'active' class on matching dot
  dots.forEach(function(d) {
    d.classList.toggle(
      'active',
      d.getAttribute('href') === '#' + current
    );
  });

}, { passive: true }); // passive = better scroll performance

The Complete Checklist

  • 1
    Set CSS Variables
    Define all colors, backgrounds, and font references in :root before writing any other CSS.
  • 2
    Apply the Universal Reset
    *, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }
  • 3
    Style the Body
    Set your background, font, font-size, line-height, and optional background textures.
  • 4
    Load Google Fonts
    Pick 1 display font, 1 body font, 1 monospace font. Link them in <head>.
  • 5
    Build the Wrapper
    max-width + margin: 0 auto centers your content at a comfortable reading width.
  • 6
    Build Reusable Components
    Cards, callout boxes, badges. Each is a CSS class you reuse throughout the page.
  • 7
    Use Pseudo-Elements for Decoration
    ::before and ::after for accent stripes, gradient lines, and overlays.
  • 8
    Add Motion
    @keyframes for entrance animations. transition for hover effects. Keep them subtle.
  • 9
    Make It Responsive
    Add @media queries to collapse multi-column grids on small screens.
  • 10
    Polish the Details
    Custom scrollbar, gradient dividers, smooth scroll, footer. These small things elevate the page from good to professional.
The Golden Rule Every design decision should serve a purpose. Colors create hierarchy. Spacing creates breathing room. Typography creates personality. Animations create life. When all of these work together under one color theme — that's when a page stops feeling like HTML and starts feeling like a product.
